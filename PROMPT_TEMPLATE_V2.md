# Tutor Skill - 分镜脚本生成提示词模板 V2

## 核心原则

### 1. TTS 可读性 (TTS Readability)

**规则：**
- ❌ **禁止 LaTeX** - TTS 引擎无法正确读取 LaTeX 公式
- ✅ **口语化描述** - 所有公式转成中文口语描述
- ✅ **<tts>标签** - 每个解释都包含口语化版本

**转换示例：**
| LaTeX | TTS 口语版 |
|-------|-----------|
| \(x^2\) | x 的平方 |
| \(\frac{a}{b}\) | b 分之 a |
| \(\sqrt{x}\) | 根号 x |
| \(\Delta\) | 判别式/德尔塔 |
| \(x_1, x_2\) | x 一 和 x 二 |
| \(-\frac{b}{2a}\) | 负 2a 分之 b |
| \(m > -\frac{1}{2}\) | m 大于负 2 分之 1 |

**语调规则：**
- 耐心、清晰、鼓励性
- 适合学生理解的口语
- 与屏幕文字意思一致

---

### 2. 教学流程 (Teaching Workflow)

#### Stage U — 理解题目
**目标：**
- 精确解析学生问题
- 检查问题是否有意义、可解
- 形式化为"已知 - 未知 - 约束 - 求解"框架

**动作：**
1. 将所有定性/视觉信息转为符号/结构化形式
2. 列出缺失或模糊信息
3. 判断：
   - 框架清晰 → 进入 Stage S
   - 模糊/矛盾 → 提出澄清问题

#### Stage S — 求解验证
**目标：**
- 私下完成完整解答（草稿纸推理）
- 验证正确性后再教学

**动作：**
1. 明确写出每一步推导，不跳步
2. 独立验证：
   - 代入检查
   - 数值/量纲/极限检查
   - 替代方法检查
3. 如验证失败：
   - 理解错误 → 返回 Stage U
   - 计算错误 → 修复后重跑 Stage S

**关键：答案必须数学正确！**

#### Stage T — 教学呈现
**目标：**
- 清晰呈现推理和结果
- 边画图边讲解
- 适当互动

**动作：**
1. 用平实语言重述问题
2. 说明方法及选择理由
3. 逐步解释推理，边画边讲
4. 总结结果和验证
5. 可选：简短拓展或洞察

**质量规则：**
- 讲解与验证答案矛盾时 → 返回 Stage S 修复
- 边画边讲，不预告画图内容
- 自然停顿，让学生消化

## 要求

### TTS 可读性要求
1. **禁止 LaTeX** - 所有公式用中文口语描述
2. **<tts>标签** - 每个分镜的读白都用<tts>包裹
3. **口语化** - 公式转成学生能听懂的中文

示例：
- 视觉文本：抛物线方程为 \(y = ax^2 + bx + c\)
- TTS 读白：<tts>抛物线方程为 y 等于 a x 平方加 b x 加 c。</tts>

### 教学流程要求

按照 U-S-T 三阶段组织内容：

**Stage U - 理解题目（约 15-20 秒）**
- 精确解析题目已知条件
- 明确求解目标
- 分析题型和考查知识点

**Stage S - 求解验证（约 60-90 秒）**
- 第 1 问：完整推导 + 验证
- 第 2 问：完整推导 + 验证
- 第 3 问：完整推导 + 验证
- 每问都要有独立检查步骤

**Stage T - 教学呈现（约 60-90 秒）**
- 清晰呈现解题思路
- 边画图边讲解
- 方法总结和拓展

## 分镜结构（8 幕）

### 第 1 幕：题目展示（Stage U，约 20-30 秒）
**画面**：深色背景，题目文字逐行显示
**字幕**：题目标题
**视觉文本**：完整题目（含 LaTeX 公式）
**TTS 读白**：<tts>口语化题目朗读</tts>
**动画**：标题淡入，方程显示，问题逐行显示
**目的**：引入题目

### 第 2 幕：条件分析（Stage U，约 15-20 秒）
**画面**：已知条件列表，求解目标高亮
**字幕**："条件分析"
**视觉文本**：已知/求解的结构化展示
**TTS 读白**：<tts>口语化分析</tts>
**动画**：条件逐条显示，关键词高亮
**目的**：理清思路

### 第 3 幕：第 1 问求解（Stage S，约 20-25 秒）
**画面**：公式推导过程
**字幕**："第 (1) 问：求顶点坐标"
**视觉文本**：LaTeX 公式推导
**TTS 读白**：<tts>口语化讲解每一步</tts>
**动画**：公式逐步显示，关键步骤高亮
**目的**：展示完整推导

### 第 4 幕：第 2 问求解（Stage S，约 25-30 秒）
**画面**：代入法和不等式求解
**字幕**："第 (2) 问：求 m 范围"
**视觉文本**：LaTeX 推导
**TTS 读白**：<tts>口语化讲解</tts>
**动画**：代入、化简、解不等式
**目的**：展示代入法

### 第 5 幕：第 3 问求解（Stage S，约 30-35 秒）
**画面**：分类讨论展示
**字幕**："第 (3) 问：最大值 - 最小值 = 4"
**视觉文本**：三种情况的 LaTeX 展示
**TTS 读白**：<tts>口语化分类讨论</tts>
**动画**：对称轴位置，三种情况逐条显示
**目的**：展示分类讨论思想

### 第 6 幕：验证答案（Stage S，约 15-20 秒）
**画面**：代入验证过程
**字幕**："验证答案"
**视觉文本**：验证步骤
**TTS 读白**：<tts>口语化验证说明</tts>
**动画**：逐问验证，显示"正确"标记
**目的**：培养验证习惯

### 第 7 幕：方法总结（Stage T，约 20-25 秒）
**画面**：知识点和方法回顾
**字幕**："方法总结"
**视觉文本**：关键公式和方法列表
**TTS 读白**：<tts>口语化总结</tts>
**动画**：要点逐条显示
**目的**：巩固提升

### 第 8 幕：拓展思考（Stage T，约 15-20 秒）
**画面**：变式问题
**字幕**："拓展思考"
**视觉文本**：变式题目
**TTS 读白**：<tts>口语化拓展</tts>
**动画**：变式问题显示，关键提示
**目的**：拓展思维

## 使用示例

### 输入
```
题目：已知抛物线 y = x² - (2m+4)x + m² + 4m
(1) 求顶点坐标
(2) A(-1,a), B(4,b) 在抛物线上，a>b，求 m 范围
(3) 在 -1≤x≤3 上，最大值 - 最小值=4，求 m

题型：抛物线综合题
难度：5 星
```

### 输出（第 3 幕示例）

```markdown
### 第 3 幕：第 1 问求解

**视觉文本**：
顶点公式：\(x = -\frac{b}{2a}\)

代入：\(x = -\frac{-(2m+4)}{2} = m+2\)

**TTS 读白**：
<tts>抛物线的顶点横坐标公式是 x 等于负 2a 分之 b。这里 a 等于 1，b 等于负 2m 减 4。代入公式，x 等于负 2 分之负 2m 减 4，化简后得到 x 等于 m 加 2。</tts>

**画面**：深色背景，公式逐步显示
**字幕**："第 (1) 问：求顶点坐标"
**动画**：
- 0.0s: 显示顶点公式
- 2.0s: 代入 a,b 值
- 4.0s: 化简得 x=m+2
- 6.0s: 高亮结果
**目的**：展示顶点坐标推导过程
```

## 优化建议

1. ** interleaving（交错呈现）**
   - 长解释分多个<tts>块
   - 文字和 TTS 交错出现
   - 学生边听边读

2. **一致性检查**
   - TTS 版与视觉文本意思一致
   - 公式转换准确
   - 无信息丢失

3. **语调控制**
   - 耐心：不催促学生
   - 清晰：每步解释到位
   - 鼓励：适当肯定

4. **自然停顿**
   - 关键步骤后停顿
   - 让学生消化
   - 不用"继续吗"等问句
